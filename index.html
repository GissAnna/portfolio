About
Software Developer & QA Engineer
Quality runs through both.

At Vakoms for over eight years, I built C++/Qt applications for demanding contexts — a tablet system monitoring medical gases in hospital wards, a cryptocurrency POS platform running on four platforms simultaneously, a real-time information display for railway transport. Projects where reliability isn't optional.

Two years ago I moved into QA engineering. Not a retreat from development — a deliberate deepening. I wanted to understand the full picture: the edge cases that only surface in production, the user flows that break on specific device firmware, the API assumptions that hold until they don't. That experience now informs how I write code.

I'm ready to return to C++ development, carrying a quality-first perspective that most developers don't have. I write code knowing how to break it — which is exactly how you write code that doesn't.
